# B-5 記事公開前 必須セルフレビューチェックリスト

> **背景**: 2026-05-17、ユーザーから「公開済み記事2本に脱字・誤字・古い値・意味不明な日本語」が報告された（[numbers-of-samurai-blue-1-2026](https://www.wc2026report.com/ja/news/numbers-of-samurai-blue-1-2026)、[netherlands-food-culture-group-f-2026](https://www.wc2026report.com/ja/news/netherlands-food-culture-group-f-2026)）。
> **目的**: Claude が `notion-create-pages` で「ステータス=公開」直接投入する前に必ずクリアすべき品質チェック手順を定義する。
> **ルール**: このチェックを通っていない記事は `ステータス=公開` で投入しない。違反した場合は即時修正。

---

## 必須運用ルール（時間軸）

### 翌日朝6時の予約公開記事は「当日の他作業に入る前」に投入する
2026-05-17、ユーザーから5/15発表記事の修正依頼を受けて作業した後、「今夕には5/18記事の下書きを投入します」と宣言したまま忘れ、5/18朝6時のcronで何も公開されない事故が発生した。

**運用ルール（厳守）:**
- ユーザーから何か依頼を受けても、**最初に Notion DBで「翌日公開予定の予約公開記事」が仕込まれているかを確認**する
- 未投入なら、依頼作業より先に翌日記事の投入を完了させる
- 投入完了をチャットで報告（タイトル・スラッグ・公開予定日）してから本来の依頼作業に移る
- ローカル scheduled-task `wcup2026-next-day-article-check`（毎日17:30 JST）でも自動チェックされるが、それを待たずに先回りする

## 必須フロー（書く順番）

### 1. 下書き投入（`ステータス=下書き`）
- `notion-create-pages` で `ステータス="下書き"` で投入する
- タグに「予約公開」を入れる場合は公開日もセット
- **「公開」での即時投入は禁止**（緊急速報を除く）

### 2. `notion-fetch` で投入結果を取得し、本文を**頭から末尾まで**読み返す
- Notion MCP のmarkdown→ブロック変換で意図しない文字化けや欠損が出ることがある
- 自動置換や中文簡体字の混入を検出する唯一の方法は、投入後本文を読み返すこと
- **5分でも読み返す**

### 3. 以下のチェック項目をすべて確認する

#### A. 文字化け・簡体字混入
過去事例で検出された簡体字（投入後に紛れ込んだ）:
- `齐 / 齑` → 正しくは `齢`（例: 「平均年齐」→「平均年齢」）
- `锁` → 正しくは `鎖`
- `丽` → 正しくは `丁` または国名（デンマーク等）
- `烊` → 正しくは `炊`（例: 「自烊」→「自炊」）

**チェック方法**: 本文を grep で「齐|齑|锁|丽|烊|鈩|龙|鏡（の異体字）」を検索

#### B. 似た漢字のtypo（変換ミス）
過去事例で検出されたtypo:
- `鍋えた` → 正しくは `鍛えた`
- `慎れた` → 正しくは `慣れた`
- `年齐` → 正しくは `年齢`

**チェック方法**: 動詞・形容詞の漢字部分をひとつずつ確認

#### C. 意味不明な日本語（造語・誤変換）
過去事例:
- `よすがにオランダ` → 正しくは `さすがにオランダ`
- `スタレチコア` → 適切な日本語に（例: `ボリュームゾーン`）
- `下肢筋を鎖上ざらにさかして` → 意味不明、削除または書き直し
- `ドレソスDF` → 接続詞のtypo、`それでDF陣` に
- `涼ようの太めポテト` → 単に `太めのポテト`

**チェック方法**: 本文を「人間が黙読」して、引っかかる箇所がないか確認

#### D. 数値の古さ（特にFIFAランク、市場価値、移籍状況）
- FIFAランクは**毎月変動**する。公開当日に最新版を確認すること
  - 確認元: https://www.fifa.com/ja/world-rankings または https://www.jsports.co.jp/football/fifa/football_men_ranking/
- 選手の所属クラブは**夏冬の移籍ウィンドウで変動**する
- 市場価値は Transfermarkt の最新値を引く
- **本文内で「YYYY-MM時点」など参照時期を明示**しておけば、後で古くなっても誠実な記事として残る

#### E. 比較対象の妥当性
過去事例:
- 「26名中22名が欧州クラブ所属。**カタール**と比べても+3名」
  - これは「カタール大会（2022年W杯）」を意図したが、読者は「カタール代表」と読み違える
  - **対策**: 比較対象は「グループF対戦相手（オランダ／チュニジア／スウェーデン）」のような具体名にする
  - または「カタール大会（2022年W杯日本代表）」と明示する

#### F. 内部リンクの整合性
- `/news/{slug}`, `/japan-squad`, `/teams/jpn` などのパスが実在するか確認
- リンク文字列が `[テキスト](URL)` の Markdown 形式になっているか
- `https://www.notion.so/...` 形式になっていないか（投入時に自動変換される場合あり）

#### G. テーブルの完全性
- `<table header-row="true">` で書いたテーブルが全行残っているか fetch で確認
- セルに `**太字**` ＋ 絵文字ショートコードを混ぜたテーブルは行が黙って消失するリスクあり

### 4. すべてクリアしたら `notion-update-page` で `ステータス="公開"` に切り替え
- もしくは「予約公開」タグを付けて翌日朝6時の自動公開cronに任せる

### 5. 公開後（ISR最大5分後）にもう一度 fetch + 本番URL確認
- 本番サイトでhtml描画されたものを `WebFetch` で見直し
- 「文字化けしていないか」「リンクが効くか」を確認

## なぜ簡体字が混入したのか？（仮説）

`notion-create-pages` MCP の本文markdown→ブロック変換で、一部の漢字が中文簡体字に自動置換されている可能性。あるいは Claude が文章生成時に頻度の低い漢字を確率で出している可能性もある。
**どちらにせよ、根本対策は「投入後の本文 fetch + 読み直し」のみ**。
